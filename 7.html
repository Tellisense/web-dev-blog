How to set up Sass in your VS Code project

What is Sass? "Sass" or "Syntactically Awesome Style Sheets" is a preprocessor for our beloved CSS. In basic terms, it gives you the capability of writing css in a more clean and streamlined manner. Then, once your done writing your super modern code, it rewrites your Sass code into simpler CSS for the browser to understand.

How do we use it? So now that we know what Sass is, how do we start using it? While there are a few methods of compiling Sass down to CSS code, this tutorial will focus primarily on the VS Code extension: Live Sass Compiler by Ritwick Dey.

File Structure. After installing Live Sass Compiler, there are only a few steps needed to get Sass up and running. First, we'll need to set up a Sass file for us to write into. Let's start by dragging an empty folder into VS Code and creating another folder named "scss." Next, create a file inside titled "style.scss" (Alternatively there is "style.sass" which is slightly different. for more info click here.) Halfway there. Now that we have our Sass location set up, we need a file for the CSS that will be generated from our Sass code. Let's create a second folder called "css" and a file inside called "style.css". Finally, we just need to link the CSS (Not the SCSS!) to our HTML file.

Watch Sass. Open your scss file and find the Watch Sass button at the bottom of your editor. Click that button. Boom Magic. Have fun writing in SCSS! If your want to learn further about this phenomenal technology, Check out the official Sass documentation!
